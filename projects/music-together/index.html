Music Together!
An upgrade
My original Spotify Party app was due for an overhaul. So, for my A-Level Computer Science NEA project I decided to work on this website. I added features like an admin interface, collaborative, persistent playlists and full playback control in the website.
How it works
Using socket.io and Node.js, I created a server for clients to talk to each other. The MySQL database would hold data, such as messages and users, inside of the database. This would be handled by the PHP before rendering a webpage to the user. An API also exists running on PHP for asynchronous requests using access tokens.
A whole journey
As this was my NEA project, I had to document everything from my research and planning to the entire development procecss and testing. This was a different approach to my usual process, but it worked out great and resulted in an well thought out project!

Strong integration
The integration with Spotify just got better! Users can search and manage songs from within the website, completely eliminating the need for the Spotify app. You can choose your own username just for the website and custom profile images.
One solution
Now you can the entire Spotify catalogue from the website and perform actions like add to queue, share and play.
Collaborative playlists
Users can now add songs together and create a playlist together, which they can access later from the Spotify app whenever they want. Users can also like and see who else liked a recently played song.
Admin interface
As the scenario for this to be used was within a school enviornment, there had to be some safeguarding features. This is why there are banned words detection and message logging. There is also a whole admin interface, with different privilege levels and in-depth user management, with the ability to ban users and see their activity.
